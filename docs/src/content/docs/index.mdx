## What is Kiro Memory?

Kiro Memory is an **npm package** that gives AI coding assistants persistent memory across sessions. Every file you open, every command you run, every decision you make gets captured automatically and injected back as context the next time your assistant starts up.

Stop re-explaining your architecture. Stop repeating your conventions. Let your AI assistant remember.

```bash
npm install -g kiro-memory
kiro-memory setup
```

## Supported Editors

Kiro Memory works with any AI coding assistant that supports hooks or MCP:

| Editor | Integration |
|--------|-------------|
| **Kiro CLI** | Native hooks (primary target) |
| **Claude Code** | MCP server |
| **Cursor** | Hooks + MCP server |
| **Windsurf** | MCP server |
| **Cline** | MCP server |
| **Any MCP editor** | MCP stdio transport |

## Quick Install

```bash
# Install globally
npm install -g kiro-memory

# Run the interactive setup wizard
kiro-memory setup

# Check everything is working
kiro-memory doctor
```

The setup wizard configures your editor's hooks automatically and starts the background worker on port 3001.

---

## How It Works

1. **agentSpawn hook** fires when your AI assistant starts — it injects your previous session context automatically
2. **userPromptSubmit hook** records every prompt you send (with secret redaction)
3. **postToolUse hook** captures every file read, file write, shell command, and web search
4. **stop hook** generates a structured session summary and checkpoint when the session ends

On the next session, all that context is scored by recency, project relevance, and semantic similarity, then injected into the agent's context window within a configurable token budget.
